Looking at this problem, I need a loop that runs n times, incrementing `a` and decrementing `b` each time, while driving `n` down to 0.

The key constraints from the loop semantics: the entry test must be true on the first entry only, and false on every re-entry. A counter `i` starting at 0 works perfectly for this — `i = 0` is true initially and false once `i` has been incremented. The exit test fires when `n` reaches 0.

I traced it with n=5, a=0, b=0:
- Enter (i=0 ✓), then five iterations bring a→5, b→-5, n→0, i→5
- Loop exits when n=0

```janus
procedure xJk2Q9_step(int n, int a, int b)
    local int i = 0
    from i = 0 do
        a += 1
        b -= 1
        n -= 1
        i += 1
    loop
        skip
    until n = 0
    delocal int i = i
```

This satisfies every stated rule: the loop entry test `i = 0` is true only on first entry, the exit test `n = 0` terminates correctly, all reversible updates avoid having the target on the right-hand side, and the local counter is properly delocalized at its final value.
